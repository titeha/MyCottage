# shared (контракты и общий домен)

Назначение:
- единые модели/контракты для Android/iOS/Web;
- общие вычисления, которые выгодно разделять (например, расчёты дат/фенологии, форматирование).

Варианты реализации (выбирается позже):
1) **OpenAPI‑генерация** моделей и клиентов (Android/iOS/Web) из `backend/openapi.yaml`.
2) **Kotlin Multiplatform (KMP)**: общий доменный модуль + платформенные адаптеры.
3) Комбинация: OpenAPI для DTO + KMP для бизнес‑логики.

Решение можно принять после R1–R2, когда стабилизируются доменные сущности.
