# Правила и события (ИИ‑наблюдения → задачи)

## 1) Задача
Сделать “умность” прикладной:
- пользователь прислал фото/вопрос,
- система распознала проблему,
- система **предложила** конкретные действия и добавила в календарь после подтверждения.

## 2) Поток данных
1) User → AI Chat (текст + фото + контекст: культура/посадка/зона)
2) AI → Observation:
- тип: вредитель/болезнь/дефицит/идентификация культуры
- объект (pest_id/disease_id/…)
- confidence
3) Rules Engine:
- сопоставить Observation с контекстом (есть ли яблоня/томаты/теплица)
- создать Issue (если нужно) и Alert
- сгенерировать предложения задач (TaskProposal)
4) User подтверждает → создаются реальные Task

## 3) Принципы безопасности
- “обработка химией” всегда требует:
  - подтверждения пользователя,
  - выбора препарата из разрешённого справочника,
  - ссылки на официальную инструкцию,
  - проверки погодных ограничений (дождь/ветер) как подсказки, а не как приказ.

## 4) Пример (совка → яблоня)
Observation: PEST=“совка”, confidence=0.78  
Контекст: есть посадки категории FRUIT_TREE (яблоня)  
Rules:
- Alert: “возможный вредитель, проверьте плоды/ловушки”
- TaskProposal:
  1) Осмотр яблонь (сегодня/завтра)
  2) Механические меры (сбор падалицы/удаление повреждённых плодов)
  3) (опционально) химия: показать список разрешённых вариантов и создать задачу только после выбора препарата
