# DEV_RULES

## Цели

- Быстрая проверка изменений.
- Минимум регрессов.
- Возможность постепенно подключать iOS и Web без переделки домена.

## Базовый принцип

API-first: контракт (OpenAPI/схемы) задаёт форму данных между Android/Web/Backend.

## Кодстайл

- Имена сущностей/классов/методов — на английском.
- Комментарии — на русском.
- Форматирование выполняем автоматически (formatter).

## Тестовая пирамида

1) Unit-тесты доменной логики (обязательно).
2) Интеграционные тесты API (по мере появления backend).
3) UI-тесты (минимум, точечно).

## Безопасность

- Секреты — только через переменные окружения/секрет-хранилище.
- Фото/данные дневника — приватные. Любая отправка в облако/ИИ только по явному согласию.

## Монетизация

Даже в MVP закладываем "права/доступ" (entitlements) и "партнерские предложения" (offers) как заглушки,
чтобы потом не ломать архитектуру.
