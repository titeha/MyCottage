# Справочник СЗР, удобрений и стимуляторов (планирование)

## 1) Зачем
Пользователи регулярно спрашивают:
- “что можно применить”, “что разрешено”, “что безопаснее”;
- “какая действующая”, “для чего подходит”, “какие ограничения”.

Цель модуля — **справочник и фильтры**, а не “волшебные рецепты”.

## 2) Источник истины
Для РФ базовый источник — **официальный реестр/каталог**, который ведётся уполномоченным органом и регулярно обновляется.

Требование ТЗ:
- хранить ссылку/версию источника (дата каталога);
- не показывать “неразрешённые” позиции;
- в карточке препарата показывать **ссылку на официальную инструкцию/регламент**.

## 3) Категории (MVP)
Пестициды (СЗР):
- инсектициды
- акарициды
- овициды (если выделяются отдельно по источнику)
- фунгициды (частый запрос)
- гербициды (частый запрос, но риск неправильного применения)
- биопрепараты (если в реестре выделены)

Агрохимикаты:
- минеральные удобрения
- органические удобрения
- органо‑минеральные
- микробиологические
- гуминовые
- мелиоранты (известковые/гипсующие и т.п.)
- стимуляторы/аминокислоты (как отдельная группа, если формально присутствует в источнике)

## 4) Safety‑политика (обязательная)
- приложение **не** генерирует “самодельные баковые смеси”;
- не даёт дозировки/регламенты “из головы”;
- любые действия показываются как “следуйте инструкции производителя и местным правилам”;
- для опасных классов — дополнительные предупреждения (пчёлы, водоёмы, СИЗ, время ожидания).

## 5) Интеграция с ИИ и задачами
Возможный сценарий:
- ИИ опознал вредителя → создал Issue → предложил план действий:
  - сначала механические/агротехнические меры,
  - затем (опционально) химия как список разрешённых вариантов из справочника,
  - задача “обработка” создаётся только после подтверждения пользователем и выбора конкретного препарата.
