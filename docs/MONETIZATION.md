# Монетизация (скелет)

## Цели

1) Не ломать приложение позже.
2) Разделить: бесплатное ядро / доп.модули / партнёрские предложения.

## Модель

- Entitlements (права): какие модули доступны пользователю.
- Offers (предложения): список товаров/сервисов партнёров, ссылки/трекинг.

## Принцип

- В коде нет "магии": всё завязано на Entitlements и FeatureGate.
- Даже если платежей нет, FeatureGate должен существовать.
- В MVP entitlements берём из локальной заглушки, позже — из backend + store receipts.

## Модули (пример)

- CORE_DIARY (ядро, всегда доступно)
- VARIETIES (справочник сортов)
- TECH_CARDS (техкарты выращивания)
- AI_CHAT (ИИ-чат)
- PRUNING (обрезка)
- DISEASES (болезни/вредители)
- SOIL_FERTILITY (восстановление плодородия)
- PLOT_PLANNER (план участка)
- RECIPES (заготовки/рецепты)
- AUTOMATION (интеграции автоматизации)
