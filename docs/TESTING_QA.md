# Тестирование и QA (v0.5)

## 1) Уровни тестов
- Unit: расчёт дат, генерация задач, агроклиматические расчёты (GDD/окна), парсинг справочников.
- Integration: Room DB, миграции, экспорт/импорт, sync‑очередь.
- E2E (Android): сценарии “добавить посадку → получить задачу → отметить → запись в журнал”.
- Backend: API contract tests, миграции БД, авторизация (OIDC), S3 media.
- Web (позже): UI e2e для планировщика и кабинета.

## 2) Особые зоны риска
- локальные уведомления (разные версии Android, ограничения батареи)
- офлайн/онлайн переходы и синхронизация (конфликты)
- работа с фото (память/разрешения/EXIF)
- корректность климатических подсказок (порог заморозков, осадки)
- безопасность AI (галлюцинации, опасные советы)
- поддержка планшетов: разные размеры/ориентации, multi‑pane навигация

## 3) Тест‑матрица устройств
Android:
- минимум 3 класса экранов: телефон (6"), планшет (8–11"), большой планшет (12"+)
- Android 9+ (или целевая политика Play Store)

Web (позже):
- Chromium/Firefox/Safari (минимум read-only в MVP веба)

## 4) Метрики качества (продукт)
- время до первой ценности (TTFV)
- доля пользователей, которые:
  - добавили ≥1 участок, ≥1 зону, ≥1 посадку
  - увидели ≥1 задачу и отметили её
- crash‑free users
- AI: полезность ответа, доля “нужны уточнения”, доля жалоб на опасные советы
