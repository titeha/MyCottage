# Приватность и безопасность (v0.5)

## 1) Чувствительные данные
- фото растений/участка;
- геолокация участка;
- заметки/журнал;
- история посадок/работ;
- данные почвенных анализов;
- (позже) интеграции автоматизации (токены/адреса).

## 2) Принципы
- Default: дневник и план участка хранятся локально; облако выключено.
- Облачная синхронизация — только после входа и явного согласия.
- AI‑обработка фото/текста — отдельное согласие (“отправлять в облако для анализа”).
- Публичность (web/форум): по умолчанию **ничего не публично**; публикация — явное действие пользователя.

## 3) Хранение и доступ
- БД: логическое разделение user data и system logs.
- Фото: object storage, доступ только через signed URLs (короткий TTL).
- Токены: short‑lived access + refresh, OIDC.
- Секреты автоматизации: шифрование + secret store в проде.

## 4) Удаление данных
- “Удалить аккаунт и данные”:
  - удаление записей из БД,
  - удаление объектов из storage,
  - аудит удаления.

## 5) AI‑логирование
- по умолчанию выключено;
- если включено — хранить минимально:
  - текст запроса (опционально, по согласию),
  - метаданные (confidence, тип наблюдения),
  - ссылки на медиа только если пользователь согласился на хранение.

## 6) Химия и безопасность
- приложение не должно подталкивать к опасному применению:
  - не генерировать “самодельные смеси”,
  - не выдавать дозировки без инструкции,
  - всегда показывать ссылку на официальный регламент/этикетку,
  - предупреждения по пчёлам/водоёмам/СИЗ/срокам ожидания.

## 7) Web/форум
- SSO через OIDC (единый аккаунт).
- раздельные политики доступа: данные дневника ≠ контент форума.
- модерация и жалобы (позже).
