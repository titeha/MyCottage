# Автоматизация (позже)

## 1) Цель
Если у пользователя есть “умные” устройства (полив, датчики), приложение может:
- показывать состояние и историю,
- выполнять расписания,
- связывать задачи дневника и погоду с действиями (например, полив).

## 2) Интеграции (предлагаемо)
- Home Assistant (как универсальный хаб)
- MQTT (как низкоуровневый протокол)

## 3) MVP подход (безопасный)
Сначала — “подсказки”:
- приложение рекомендует полив/проветривание, но не управляет.

Далее — управление:
- пользователь связывает задачу “полив” с конкретным устройством/сценарием
- управление выполняется только с подтверждением и логируется.

## 4) Безопасность
- хранение токенов/ключей зашифровано
- журнал действий
- возможность “отключить автоматизацию” одним переключателем
