# Моя дача (Mobile + Web + Cloud)

Проект: офлайн‑дневник дачника + облачная платформа (синхронизация, ИИ, погода/агроклимат, почвы, справочники) + веб‑портал (сообщество и “компьютерные” инструменты).

Дата шаблона: 2026-02-22

## 1) Платформы

- **Android** — основной клиент (старт разработки).
- **iOS (позже)** — архитектура готовится заранее (API‑first + общий доменный слой/контракты).
- **Планшеты** — обязательная поддержка (адаптивная UI‑компоновка).
- **Web** — портал: форум/соцсеть + инструменты (план участка, большие таблицы, аналитика, импорт/экспорт).

## 2) Коротко о продукте

### Ядро (обязательное)
1) **Дневник дачника (офлайн‑first)**
- участки и зоны (грядки/теплица/сад/цветник);
- посадки/растения: что, где и когда;
- журнал работ (заметки + фото);
- задачи/календарь/напоминания.
- **диаграмма сезона (Timeline/Gantt)**: посадки → рост → созревание → сбор (наглядно).

2) **Климат и базовые расчёты сроков**
- профиль участка: регион/координаты, окна сезона (последний/первый заморозок как оценка);
- погодный слой (если есть интернет): риск заморозков, осадки, базовые подсказки;
- расчёт “окна сбора урожая”/первого сбора (ориентировочно).

3) **План участка с посадками (Plot Planner v1)**
- схема участка/зон;
- размещение посадок на схеме;
- визуальная проверка “что где растёт”.

### Расширения (поэтапно)
- **Техкарты выращивания культур** (шаблоны ухода по культурам/сортам + генерация задач).
- **Справочник сортов** (карточки сортов, сроки, особенности, регионы допуска).
- **Почвы** (профиль, почвенная карта, лабораторные анализы) + **восстановление плодородия**.
- **Справочник СЗР/удобрений** (только разрешённые и с дисклеймерами; без “самодеятельных” рецептов).
- **ИИ‑помощник** (чат + подсказки по дневнику) и **ИИ‑наблюдения → задачи**:
  - фото → распознавание (вредитель/болезнь/дефицит/культура) → предупреждение → предложения задач.
- **Обрезка кустов/деревьев по фото/видео** (поэтапно: от “инструкции” к “подсветке зон”).
- **Заготовки/рецепты** (предложения “что приготовить/законсервировать” при сборе урожая).
- **Автоматизация** (интеграция с Home Assistant/MQTT, управление поливом/режимами по расписанию и погоде).
- **Магазин** (сначала список покупок + партнёрские ссылки; потом полноценная корзина).

## 3) Репозиторий: структура

- `android-app/` — Android‑приложение (Kotlin; рекомендуемо Jetpack Compose + Room).
- `ios-app/` — заготовка под iOS (позже; контракты/плейсхолдер).
- `shared/` — общий доменный слой/контракты (вариант: Kotlin Multiplatform или OpenAPI‑генерация моделей).
- `web-portal/` — веб‑портал (форум/сообщество + инструменты, позже).
- `backend/` — API и сервисы (скелет; язык/фреймворк выбирается).
- `infra/` — инфраструктура (docker compose для dev, шаблоны IaC).
- `docs/` — ТЗ, архитектура, API, модель данных, roadmap.
- `scripts/` — утилиты, миграции, генераторы справочников.
- `data/` — примерные справочники/схемы (без персональных данных).

## 4) Быстрый старт (dev)

Репозиторий — каркас. Для локальной разработки облачной части используйте `infra/docker-compose.yml`.

1) Установить Docker + Docker Compose
2) Запустить:
```bash
cd infra
docker compose up -d
```
