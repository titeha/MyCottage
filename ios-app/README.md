# iOS (placeholder)

Цель: заранее держать архитектуру готовой к iOS без необходимости “перепридумывать” домен и API.

Подход:
- API‑first: мобильные клиенты общаются с Cloud API по единым контрактам.
- Общие модели данных/DTO генерируются из OpenAPI (`backend/openapi.yaml`) или поддерживаются в `shared/`.
- Общая бизнес‑логика (опционально) выносится в `shared/` (например, Kotlin Multiplatform).
- UI для iOS планируется отдельным слоем (SwiftUI), когда появится возможность собирать и тестировать.

На старте папка служит как место для:
- описания будущего стека,
- прототипов,
- CI‑пайплайнов и инструкций.
