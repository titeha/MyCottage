# Web Portal (placeholder)

Задачи веб‑части:
- маркетинговый сайт (лендинг, FAQ);
- сообщество: форум/вопросы‑ответы/профили участков (позже);
- инструменты, которые удобнее на компьютере:
  - план участка (большой экран, drag&drop),
  - импорт/экспорт и массовые правки,
  - аналитика сезона (графики погоды/урожая),
  - “авто‑рассадка/размещение” (экспериментально, AI).

Стартовый путь (рекомендуемо):
- форум как готовый движок (например, Discourse) + SSO (Keycloak),
- отдельное SPA (например Next.js) для планировщика и кабинета.

На старте папка служит как место для:
- структуры веб‑проекта,
- CI,
- общих компонентов/дизайн‑токенов.
